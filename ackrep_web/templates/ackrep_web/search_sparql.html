{% endif %} Help and Examples for SPARQL Syntax: [ / ] SPARQL queries can be read as a very simple sentence of subject, predicate (sometimes a auxiliary verb such as "has" or "is" is implied), object. E.g. SELECT ?book WHERE { ?book has_author MarkTwain. } selects all books of the author Mark Twain existing in the database. Depending on where the subject, predicate or object is defined, different prefixes have to be prepended (e.g. "rdf:", "rdfs:" for general namespaces and "P:" for the ackrep-specific namespace). For a more detailed introduction to SPARQL Queries, see Wikidata SPARQL tutorial.

Some examples of usefull queries in the ackrep context:
Find the entity with a given key (e.g. UXMFA): PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?entity WHERE { ?entity ocse:R2950 "UXMFA". } or if the label is known, but the short key (R2950) isn't: PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?entity WHERE { ?entity ?relation "UXMFA". ?relation :R1 "has corresponding ackrep key". }
Get all relations of all exising system models: PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?relation WHERE { ?entity ?relation ?o. ?entity :R4 ocse:I7641. } Get all relations that have a system model as first argument: PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?relation WHERE { ?relation :R8 ocse:I7641. } Note the difference between the two preceding queries!

Get all pyerk items: PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?item WHERE { ?item (:R4|:R3)* :I1 }
Some more useful and complex queries:
Find all system models that are represented by a partial differential equation: PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?entity WHERE { ?entity :R4 ocse:I7641. ?entity ocse:R2928 ?representation. ?representation :R4 ocse:I8063. }
Find all system models that have system order of 3 and are autonomous PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?entity WHERE { ?entity :R4 ocse:I7641. ?entity ocse:R2928 ?r. ?r ocse:R3898 3. ?entity :R16 ocse:I5718. }
Note: Tags of problems and solutions are depricated and will soon be implemented in pyerk.
Select all ackrep entities that use linear state feedback: PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?entity ?tag WHERE { ?entity ocse:R1070 ?tag. ?tag :R1 "ocse:Linear_State_Feedback". }
Select all entites, that have both of the tags "Convergence_To_Desired_Trajectory" and "Trajectory_Generation": PREFIX : <erk:/builtins#> PREFIX ocse: <erk:/ocse/0.2#> SELECT ?entity WHERE { ?entity ocse:R1070 ?tag1. ?entity ocse:R1070 ?tag2. ?tag1 :R1 "ocse:Convergence_To_Desired_Trajectory". ?tag2 :R1 "ocse:Trajectory_Generation". }
